# 자기소개 홈페이지 PRD

> **문서 버전** v1.1 · **작성일** 2026-06-30 · **작성자** 전혜림
> **목적** 채용 담당자가 전혜림의 자기소개와 포트폴리오를 한눈에 확인하고 채용 적합성을 검증할 수 있는 1인 개발자 소개 홈페이지(싱글 페이지 + 섹션 네비게이션) 구축

---

## 1. 개요 (Overview)

채용 담당자를 핵심 사용자로 하는 **자기소개·포트폴리오 홈페이지**를 제작한다. 백엔드 개발자 전혜림의 역량, 프로젝트 경험, 연락 수단을 명확하고 매력적으로 전달하여, 짧은 시간 안에 "면접에 부를 만한 후보인가"를 판단할 수 있도록 돕는 것이 목표다.

부드러운 스크롤, 패럴랙스, 마우스 인터랙션(잔상/파티클) 등 절제된 모션을 통해 프론트엔드 감각과 디테일에 대한 신뢰를 함께 전달한다.

---

## 2. 배경 및 문제 정의

- 채용 담당자는 다수의 지원자를 짧은 시간에 검토하며, 정보가 흩어진 이력서·링크는 평가 효율을 떨어뜨린다.
- 정적인 PDF 이력서만으로는 지원자의 실제 결과물(화면, 성과 수치)과 개발 감각을 전달하기 어렵다.
- 따라서 **핵심 정보 → 깊이 있는 근거**로 자연스럽게 흐르는 단일 진입점 홈페이지가 필요하다.

---

## 3. 목표 및 성공 지표

### 3.1 목표
- 채용 담당자가 **3분 이내**에 지원자의 핵심 역량·대표 프로젝트·연락처를 파악할 수 있다.
- 대표 프로젝트의 **정량 성과**(예: TPS 50% 향상)를 즉시 확인할 수 있다.
- 이메일/GitHub로의 전환(연락·코드 확인)이 자연스럽게 유도된다.

### 3.2 성공 지표 (정성/정량)
| 지표 | 목표 |
|------|------|
| 핵심 정보 도달 시간 | 첫 화면 진입 후 3분 이내 |
| 연락처/GitHub 클릭률 | 방문 세션 대비 명확한 CTA 노출 |
| 모바일/데스크톱 정상 표시 | 주요 브라우저·해상도 100% |
| 페이지 로드(LCP) | 2.5초 이내 |

## 5. 정보 구조 (IA) & 메뉴 구성

전역 상단 고정 네비게이션(GNB)에서 각 섹션으로 부드럽게 스크롤 이동하는 **싱글 페이지** 구조를 기본으로 한다.

| 메뉴 | 섹션 ID | 역할 |
|------|---------|------|
| **홈** | `#home` | 첫인상. 이름, 한 줄 슬로건, 핵심 키워드, 메인 CTA |
| **포트폴리오** | `#portfolio` | 대표 프로젝트 카드/갤러리 + 성과·기술 |
| **자기소개** | `#about` | 개발 철학, 핵심 역량, 학력, 스킬, 수상/자격증 |
| **연락처** | `#contact` | 이메일, GitHub (필요 시 전화) |

- GNB는 스크롤 시에도 상단 고정(sticky), 현재 위치한 섹션을 하이라이트(scroll-spy).
- 모바일에서는 햄버거 메뉴로 전환.

## 7. 애니메이션 / 인터랙션 요구사항

> 모션은 "과하지 않게, 콘텐츠를 보조"하는 수준으로. 성능과 접근성을 우선한다.

1. **부드러운 스크롤 (Smooth Scroll)**
   - GNB 메뉴 클릭 시 해당 섹션으로 부드럽게 스크롤 이동.
   - 전역 스크롤 감속/관성 처리(예: Lenis 등 경량 라이브러리 또는 CSS `scroll-behavior`).
2. **패럴랙스 스크롤 (Parallax)**
   - Hero 배경·섹션 구분 영역에서 깊이감 있는 다층 스크롤.
   - 스크롤 진입 시 섹션 요소 fade/slide-in(Intersection Observer 기반).
3. **마우스 추적 효과 (잔상 / 파티클)**
   - 커서를 따라다니는 파티클 또는 트레일 잔상 효과(주로 Hero 영역).
   - 성능 보호를 위해 모바일/터치 환경에서는 비활성화, 프레임 최적화(requestAnimationFrame, 입자 수 상한).

**공통 모션 원칙**
- `prefers-reduced-motion` 설정 존중 → 모션 최소화 모드 제공.
- 60fps 목표, 메인 스레드 블로킹 금지(가능 시 transform/opacity 기반 GPU 가속).

## 8. 비기능 요구사항 (Non-functional)

- **반응형**: 모바일(≤768px)·태블릿·데스크톱 대응. 모바일 우선 레이아웃 검증.
- **성능**: LCP ≤ 2.5s, 이미지 최적화(WebP 변환·lazy load), 모션 60fps.
- **접근성(A11y)**: 시맨틱 마크업, 키보드 내비게이션, 색 대비 WCAG AA, `prefers-reduced-motion` 대응, 이미지 `alt`.
- **SEO/공유**: 메타 태그, Open Graph(프로필 썸네일), 타이틀·설명.
- **브라우저 지원**: 최신 Chrome, Safari, Edge, Firefox.

## 10. 디자인 가이드 (톤앤매너)

- **인상**: 신뢰감 있는 백엔드 개발자 + 정돈된 디테일. 미니멀·여백 충분·가독성 우선.
- **컬러**: 차분한 베이스(화이트/딥네이비·그레이) + 1개의 포인트 컬러(액션·강조).
- **타이포**: 한글 가독성 좋은 산세리프(예: Pretendard). 위계(제목/본문/캡션) 명확.
- **콘텐츠 우선**: 성과 수치는 크게·강조, 장식 모션은 보조.
